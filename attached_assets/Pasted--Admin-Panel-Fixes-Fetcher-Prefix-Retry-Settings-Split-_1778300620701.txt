 Admin Panel Fixes — Fetcher Prefix, Retry, Settings Split, as-any
What & Why
Four confirmed bugs and quality gaps in the admin panel: (1) dashboard.tsx calls a plain fetcher() instead of adminFetcher() for the export endpoint, bypassing admin authentication; (2) React Query retry: false globally means any transient error shows a permanent error state; (3) settings.tsx is still 1152 lines with inline JSX for General, Notifications, and Monitoring tabs; (4) 21 as any casts across admin components mask type errors.

Done looks like
dashboard.tsx export call uses adminFetcher("/fleet/dashboard-export") (authenticated) instead of the unauthenticated fetcher()
Global React Query config is updated from retry: false to retry: 1 (one automatic retry on transient errors) with retryDelay: 1000 (1s between retries)
The three remaining inline tab sections in settings.tsx — General, Notifications, Monitoring — are extracted into settings-general.tsx, settings-notifications.tsx, and settings-monitoring.tsx following the exact pattern of the existing settings-payment.tsx, settings-integrations.tsx, and settings-security.tsx files. settings.tsx becomes a thin router/orchestrator.
as any casts in admin source are reduced by at least 70% (below 7). The most impactful are in hooks/use-admin.ts response parsing and lib/adminFetcher.tsx error handling.
